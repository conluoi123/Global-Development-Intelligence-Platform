Design Principles — Trước khi nhìn vào diagram

Senior Engineer thiết kế cho failure, không phải cho happy path. Mỗi quyết định kỹ thuật trong GDIP đều có thể justify bằng một trong 6 nguyên tắc sau:

PrincipleVấn đề nó giải quyếtÁp dụng trong GDIP
Idempotency Pipeline re-run tạo duplicate data fetch_id = sha256(params) → check manifest trước khi fetch. DAG re-run 100 lần = 1 lần.
Fail-fast Bad data lan vào Gold, khó debug sau Great Expectations CRITICAL gate chặn tại Bronze→Silver. Bad data không bao giờ vào Gold.
Immutability Bronze bị overwrite → mất audit trail Delta table Bronze: appendOnly=true. Mọi fetch đều preserved forever.
Schema Evolution WB thêm column mới → pipeline crash dbt on_schema_change='append_new_columns'. API response schema versioned /v1/ /v2/.
Observability-first Bug production không debug được vì thiếu log Structured JSON log mọi request. OpenTelemetry traces. Prometheus metrics từ ngày 1.
Interface Contracts AI model query Bronze trực tiếp → coupling AI chỉ đọc Gold qua defined interfaces. Schema version + SLA documented.

Idempotency & Error Handling
🔐
Idempotency Pattern
DE Core
fetch_id = sha256(country + indicator + date_range + api_version)

Mỗi ingest run check manifest trước. Nếu fetch_id đã tồn tại → skip. DAG có thể re-run bao nhiêu lần cũng được mà không tạo duplicate.
⚡
Retry & Circuit Breaker
Resilience
  • 429 Rate limit → exponential backoff: 1s, 2s, 4s, 8s
  • 504 Timeout → chunk date range nhỏ hơn, retry
  • >10 consecutive failures → Circuit breaker OPEN → PagerDuty
  • Schema change → HALT + alert (không retry)
💧
Incremental Loading
Efficiency
Watermark table track last_fetched_year per indicator. Chỉ fetch data mới hơn watermark. Kết hợp với manifest check để đảm bảo không bao giờ có gap hoặc duplicate.
🚦
Quality Gates (3 Levels)
Data Quality
  • CRITICAL → Pipeline HALT, Slack alert
  • WARNING → Log + alert, pipeline tiếp tục
  • INFO → Log only
  • Row count drop >5% → automatic HALT
AI Engineering — System Design
Model Versioning (2000+ Models)
📦
MLflow Registry Architecture
Model Management
200 countries × 10+ indicators = 2000+ models. Không thể track thủ công.

Experiment structure: gdip/forecasting/{indicator_code}
Model lifecycle: None → Staging → Production → Archived
Auto-promote nếu: MAPE holdout không tệ hơn current Production > 5%, coverage_90 > 88%, latency p95 < 500ms
Auto-rollback: Nếu drift detected → rollback < 5 phút, alert on-call
Evaluation Framework
📊
Walk-Forward CV (Không Random Split)
Time-Series
Random split = look-ahead bias = sai lầm nghiêm trọng nhất.

Walk-forward: train [1960–2010] → validate [2011–2013]
Slide forward 4 lần. Holdout [2021–2023] chỉ dùng 1 lần duy nhất.
std_mape cao → model không stable → red flag.
🎯
Anomaly Calibration
Threshold
Ground truth = historical crises (2008, 1997 Asian, 2020 COVID).

Optimize threshold: maximize F1 subject to FPR < 5%.
Hard constraint trước, accuracy sau. FPR cao = alert fatigue = engineer ignore alert.
📡
Production Drift Monitoring
MLOps
  • Data drift: KS test daily, PSI per feature
  • Prediction drift: distribution shift weekly
  • Ground truth: compare vs WB actual khi có
  • Staleness: >90 ngày → force retrain
💬
RAG Evaluation (RAGAS)
NLP
Eval dataset: 50 QA pairs = 20 manual + 20 LLM-generated + 10 adversarial.

Adversarial = câu hỏi không có answer trong corpus. Đo hallucination rate và abstention behavior. Target: hallucination < 5%.
Model Serving Architecture

Interface Contracts — DE ↔ AIE Boundary

AI models không được query Bronze/Silver trực tiếp. Mọi AI consumption phải qua defined Gold interfaces. Đây là ranh giới quan trọng nhất giữa DE và AIE ownership.

01 Feature Store Interface
Table: gold.feature_store — versioned, schema v3
SLA: Data available by 08:00 UTC sau mỗi DAG run
Breaking change protocol: 2-week notice + versioned table song song
Owned by: DE team. AI team là consumer, không modify.
02 Model Input Specification
Mỗi model có input_schema.json explicit:

{"features": ["value_lag_1y", "rolling_avg_5y"], "min_history_required": 10}

Nếu entity có < 10 năm data: model trả về null + warning, không crash. Caller phải handle warning.
03 API Versioning Policy
Endpoint versioned: /v1/, /v2/ — consumers phải specify version
Deprecation: v(n-1) supported 6 tháng sau v(n) release
Breaking change: yêu cầu major version bump + migration guide
Non-breaking: thêm optional field = không cần version bump
04 SQL Safety Contract (Text-to-SQL)
Allowed: SELECT only, max 1000 rows, timeout 10s
Forbidden: INSERT, UPDATE, DELETE, DROP, CREATE, pg_sleep, UNION injection
Validation: SQL parsed và validated TRƯỚC khi execute — 100% enforced
Exposed schema: Chỉ Gold tables, không expose Bronze/Silver hoặc audit tables

Failure Modes — Hệ thống fail như thế nào?

Senior Engineer thiết kế bằng cách enumerate mọi cách hệ thống có thể fail, sau đó mitigation từng cái. Không phải sau khi fail mới fix.

FailureImpactDetectionMitigation
WB API down HIGH Airflow task failure alert Circuit breaker → fallback cached Bronze, retry queue, alert on-call
WB schema change HIGH APISchemaChangedError exception HALT pipeline, Slack alert, manifest.mark_failed → human review required
Bad data into pipeline HIGH Great Expectations CRITICAL gate Fail-fast ở Bronze→Silver, không bao giờ vào Gold, detailed error log
Row count giảm bất thường MEDIUM dbt test row_count_vs_previous_run Auto-halt nếu >5% drop, alert team để investigate
Model performance degraded MEDIUM Drift detector, ground truth validation Auto-rollback <5 phút, alert, flag cho manual review
LLM API down (OpenAI) MEDIUM Circuit breaker + health check Fallback sang smaller local model, graceful degradation response
RAG hallucination spike MEDIUM RAGAS hallucination rate monitoring Alert + disable RAG endpoint, fallback to keyword-only search
Gold data stale (>26h) HIGH Airflow SLA miss alert PagerDuty alert, API returns stale_data warning in response
Storage near full CRITICAL MinIO capacity metric in Grafana Alert at 80%, auto-archive Bronze >3 years, block writes at 95%
SQL injection via Text-to-SQL CRITICAL SQL validator trước execute Reject non-SELECT, parameterized queries, isolated read-only user
